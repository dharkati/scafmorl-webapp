# ScafMORL — ERα Drug Explorer (client-side web app)

A static, **fully client-side** (HTML/CSS/JS only, no server) explorer for
molecules generated by ScafMORL, deployable on GitHub Pages.

It reproduces the display of the original Flask app (`server.py` in the main
code repository) — predicted ERα pIC50, physicochemical properties, ADME/hERG
profile, drug-likeness filters, and Tanimoto similarity to approved/clinical ERα
ligands — but instead of running the neural generator and QSAR model on a
server, it **samples from a pool of 1,000 molecules pre-generated by the ScafMORL
model** (a reproducible random subset of the published 10,000-molecule library),
with all properties precomputed offline. Structures are drawn in the browser with
[SmilesDrawer](https://github.com/reymond-group/smilesDrawer).

## Files

| File | Purpose |
|------|---------|
| `index.html` | Page structure |
| `style.css`  | Styling (dark theme) |
| `app.js`     | Loads `molecules.json`, cycles through molecules on click, renders |
| `molecules.json` | 1,000 pre-generated molecules with full properties + per-molecule top-6 similar approved ligands |
| `molecules_1000.csv` | The same 1,000 molecules as a flat CSV (properties only; no nested similarity) |

## Run locally

Because the app fetches `molecules.json`, it must be served over HTTP (opening
`index.html` directly via `file://` is blocked by the browser). Any static
server works:

```bash
cd scafmorl_webapp
python -m http.server 8000
# open http://localhost:8000
```

## Deploy on GitHub Pages

1. Put the contents of this folder at the root of a repository (e.g.
   `github.com/dharkati/scafmorl-webapp`).
2. Repo **Settings → Pages → Build and deployment → Source: Deploy from a
   branch**, branch `main`, folder `/ (root)`.
3. The app will be live at `https://dharkati.github.io/scafmorl-webapp/`.

No build step is required; everything is static.

## Note

This is a demonstration/exploration tool. Predicted pIC50 comes from an XGBoost
QSAR surrogate (R² = 0.778, 5-fold CV) and the ADME/CYP/hERG values are
rule-based structural alerts — see the paper for their limitations. The tool
does not perform live neural generation; regenerate `molecules.json` from the
published library with `build_1000_dataset.py` (in the main code repository) if
you want a different sample.
